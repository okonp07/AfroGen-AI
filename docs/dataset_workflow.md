# Dataset Workflow

## Goal

Prepare a reproducible training manifest for afrocentric portrait generation.

## Folder Layout

- `data/raw/`
  Store source images here in class or batch folders.
- `data/raw/metadata.csv`
  Optional metadata sheet with one row per image.
- `data/raw/metadata_template.csv`
  Starter template you can copy into `metadata.csv`.
- `data/processed/manifest.jsonl`
  Canonical training manifest generated by the prep script.

## Supported Metadata Columns

The prep script understands these columns:

- `file_name`
- `prompt`
- `gender_presentation`
- `age_group`
- `skin_tone`
- `hairstyle`
- `expression`
- `accessory`
- `lighting`
- `background`
- `split`
- `source`
- `tags`

Only `file_name` is required when using `metadata.csv`.

## How To Build The Manifest

```bash
python3 scripts/prepare_dataset.py
```

This will:

- scan `data/raw/` for images
- merge image paths with optional metadata
- generate default prompts when missing
- write `data/processed/manifest.jsonl`
- print a short dataset summary

## Why The Manifest Matters

The manifest gives us a stable handoff between data collection and training:

- it is versionable
- it is scriptable
- it can feed diffusion, GAN, or hybrid pipelines later
